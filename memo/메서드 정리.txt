
클래스 메서드와 인스턴드 메서드의 차이

iv = 인스턴스 변수(변수)
lv = 클래스 변수(지역변수)

인스턴스 멤버 (iv, im)를 사용하지 않는 메서드에 static(클래스 인스턴스)을 붙인다

[클래스 메서드]
클래스 메서드는 객체 생성 없이 사용할 수 있는 메서드를 의미한다
클래스이름.메서드이름(지역변수) 로 사용할 수 있다

static int add(int a, int b)
    return a + b;


* iv가 필요하지 않다 또한 쓸수없다 이는 객체가 있는지 없는지 알 수 없어서이다.
* 클래스 메서드와 클래스 메서드 끼리는 호출이 가능하다
* 클래스 메서드는 인스턴스 변수를 사용할 수 없다 이유는 객체를 생성해야지 사용할 수 있기 때문이다
* 클래스 메서드는 인스턴스 메서드를 호출할 수 없다

[인스턴스 메서드]
인스턴스 내부에서 객체를 생성하여 사용하는 메서드를 의미한다

java j = new java();

j.a = 10
j.b = 10

int add()
    return a + b;

System.out.println(j.add());

* iv가 필요하다

[오버로딩]

한클래스안에 같은 이름의 메서드를 여러개 정의하는것

* 메서드 이름이 같아야한다
* 매개변수 개수 또는 타입이 달라야 한다
* 변환 타입의 영향은 없다
